ring-i18n Component Demo

ring-i18n — internationalization service element for the Nürburgring components. Ships with English and German bundles (section names and UI labels), detects the browser language on connect (overridable via the lang attribute), and translates dot-separated keys with {param} substitution.

On every language change it re-translates all elements carrying a data-i18n="<key>" attribute in the document, notifies registered observers, and reflects the language in the lang attribute.

It renders nothing — place it once per page:

<ring-i18n lang="de"></ring-i18n>
<h1 data-i18n="ui.trackMap"></h1>

Methods

MethodReturnsDescription
getCurrentLanguage() => stringThe current language code.
getLanguages() => string[]Language codes with a registered bundle.
setLanguage(language: string) => voidSwitch the current language. Accepts BCP-47 / cased tags ('de-DE', 'DE' → de); unresolvable languages are ignored. Reflects the canonical code in the lang attribute, re-translates [data-i18n] elements, notifies observers, and emits language-changed.
t(key: string, params?: Record<string, string | number>) => stringTranslate a dot-separated key (e.g. 'sections.Fuchsröhre' or 'ui.totalDistance') with {param} substitution. Falls back to English, then to the key itself. (Named t because translate is an inherited HTMLElement boolean property.)
translateKey(key: string, params?: Record<string, string | number>) => stringAlias for {@link t}.
addTranslations(language: string, bundle: Partial<TranslationBundle>) => voidRegister (or extend) the bundle for a language. Existing keys for the language are kept unless overridden.
registerObserver(callback: LanguageObserver) => voidRegister an observer called with the language code on every change.
unregisterObserver(callback: LanguageObserver) => voidRemove a previously registered observer.
applyTranslations(root?: ParentNode) => voidTranslate every element with a data-i18n="<key>" attribute under root (the whole document by default) by setting its text content.
